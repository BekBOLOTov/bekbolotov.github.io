# Requirements — CV Landing Page

**Milestone:** v1.0
**Status:** Scoped

---

## v1 Requirements

### Hero / Identity

- [ ] **HERO-01**: Visitor sees name and job title prominently on page load
- [ ] **HERO-02**: Hero section contains a 2–3 sentence specific summary with domain, scale, or specialization signal (not generic)
- [ ] **HERO-03**: At least one contact or profile link (email, GitHub) is visible above the fold without scrolling

### Work Experience

- [ ] **EXP-01**: Visitor can scan reverse-chronological work history — company, title, and dates for each role
- [ ] **EXP-02**: Each role has at least one impact-framed bullet point (problem → action → result format, quantified where possible)
- [ ] **EXP-03**: Each role lists the tech stack and tools used

### Skills

- [ ] **SKILL-01**: Skills are grouped by honest proficiency tier (e.g., Expert / Proficient / Familiar) — not a flat keyword dump

### Achievements

- [ ] **EDU-01**: Achievements section lists competitive programming honors — IOI 2016/2017, ACM ICPC NEERC 2018, and DS&A problem-solving profiles (Codeforces, LeetCode, ACMP)

### Contact / Links

- [ ] **CONTACT-01**: Contact section includes email, GitHub profile link, and LinkedIn profile link

### Delivery / Quality

- [ ] **DEPLOY-01**: Site is live and publicly accessible on GitHub Pages (username.github.io)
- [ ] **POLISH-01**: Page renders correctly on mobile at 375px width — single-column, readable, no overflow
- [ ] **POLISH-02**: Browser "Print to PDF" produces a clean, readable CV document (`@media print` stylesheet)
- [ ] **POLISH-03**: Page has Open Graph meta tags and a `<title>`/`<meta description>` tuned for name search and LinkedIn link previews

---

## v2 Requirements (Deferred)

- Projects section — showcase 2–4 projects with links and descriptions
- Certifications (if any added later)
- Favicon
- Custom domain (`CNAME` + DNS)
- Dark mode toggle
- Blog / writing section

---

## Out of Scope

- Backend or CMS — static only, content hardcoded in HTML
- Contact form — links only, no server-side form handling
- JavaScript animations, scroll effects, typed text — compete with readability
- Skills progress bars — visually misleading, undercut credibility signal
- Multiple pages or routing — single scrollable page only
