<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| index.html <head> → web crawlers / social scrapers | OG tags and JSON-LD are read by LinkedIn, Slack, Google crawlers — content is hardcoded, no user input |
| JSON-LD script block → browser JS engine | script type="application/ld+json" is parsed as data, not executed as code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | JSON-LD script block — could be misread as executable JS | accept | script type="application/ld+json" is treated as data by browsers; it is not executed. Content is hardcoded with no user-supplied interpolation. Validated as JSON in acceptance criteria. |
| T-03-03-02 | Information Disclosure | JSON-LD email field "mailto:bekbolotov.bolot@gmail.com" | accept | The email is already visible in the hero and contact sections of index.html. Including it in JSON-LD adds no new exposure. This is deliberate on a public CV. |
| T-03-03-03 | Information Disclosure | JSON-LD sameAs array — public profile URLs (GitHub, LinkedIn, Codeforces, etc.) | accept | All URLs are intentionally public profile links chosen by the engineer for the CV. No private data. |
| T-03-03-04 | Elevation of Privilege | og:url set to https://bolotbekbolotov.github.io/ | accept | The URL is the canonical site URL (locked by D-23). No open redirect vector — the URL is hardcoded to the site itself, not to user-controlled input. |
| T-03-03-05 | Information Disclosure | og:image absolute URL pointing to assets/og-card.png | accept | The PNG is a public static asset on GitHub Pages (HTTPS). No user data in the image. |
</threat_model>
