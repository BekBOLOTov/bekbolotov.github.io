**Step 1 — Enable GitHub Pages from main branch root:**

Primary path (curl + `$GITHUB_TOKEN`):
```bash
curl -L \
  -X POST \
  -H "Accept: application/vnd.github+json" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  https://api.github.com/repos/bolotbekbolotov/bolotbekbolotov.github.io/pages \
  -d '{"source":{"branch":"main","path":"/"}}'
```

Expected response: JSON containing `"url": "https://api.github.com/repos/bolotbekbolotov/bolotbekbolotov.github.io/pages"` and `"status": "queued"` or `"built"`.

If the API returns HTTP 409 ("Pages is already enabled"), proceed to Step 2 — Pages may have been auto-enabled for a username.github.io repo.

Fallback (no token): GitHub → Repository → Settings → Pages → Source: "Deploy from a branch" → Branch: `main` / Folder: `/ (root)` → Save.

**Step 2 — Poll for HTTPS availability (up to 20 minutes):**

Run the following polling loop. It checks every 60 seconds and exits when HTTPS is confirmed:

```bash
MAX_WAIT=1200  # 20 minutes in seconds
INTERVAL=60
ELAPSED=0

while [ $ELAPSED -lt $MAX_WAIT ]; do
  STATUS=$(curl -s \
    -H "Accept: application/vnd.github+json" \
    -H "Authorization: Bearer $GITHUB_TOKEN" \
    https://api.github.com/repos/bolotbekbolotov/bolotbekbolotov.github.io/pages \
    | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('https_enforced','false'))" 2>/dev/null)

  HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" https://bolotbekbolotov.github.io)

  echo "[${ELAPSED}s] https_enforced=${STATUS}, HTTP response=${HTTP_CODE}"

  if [ "$STATUS" = "True" ] && [ "$HTTP_CODE" = "200" ]; then
    echo "HTTPS confirmed live."
    break
  fi

  sleep $INTERVAL
  ELAPSED=$((ELAPSED + INTERVAL))
done

if [ $ELAPSED -ge $MAX_WAIT ]; then
  echo "TIMEOUT: HTTPS not confirmed after 20 minutes. Check GitHub Settings → Pages and GitHub status page."
fi
```

If `$GITHUB_TOKEN` is not set, poll manually: open https://github.com/bolotbekbolotov/bolotbekbolotov.github.io/settings/pages every 2 minutes until the "Enforce HTTPS" checkbox is checked and the URL shows "Your site is live at https://bolotbekbolotov.github.io".

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curl → GitHub REST API | GITHUB_TOKEN transmitted in Authorization header; must use HTTPS only |
| git push → github.com | Credentials transmitted via HTTPS remote URL |
| Visitor → GitHub Pages CDN | Browser requests served over HTTPS with Let's Encrypt cert |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | GITHUB_TOKEN in shell environment | mitigate | Token used only in curl commands; never written to any file in the repo. After deploy, token can be revoked or scoped down. Never commit token to index.html, style.css, or any tracked file. |
| T-02-02 | Information Disclosure | HTTP → HTTPS redirect (before cert is provisioned) | mitigate | Do not share the URL until Task 2 HTTPS verification passes. "Enforce HTTPS" checkbox must be confirmed checked before handing off to human verify checkpoint. |
| T-02-03 | Tampering | git remote URL with token embedded | mitigate | If token is embedded in remote URL for push auth (`https://$TOKEN@github.com/...`), reset remote URL to tokenless form after push: `git remote set-url origin https://github.com/bolotbekbolotov/bolotbekbolotov.github.io.git` |
| T-02-04 | Denial of Service | GitHub CDN availability | accept | GitHub Pages CDN availability is GitHub's responsibility. No mitigation available at this tier. |
</threat_model>
